## الجزء الأول — معالج إضافة عميل `/admin/dealers/new` (المرجع الذهبي 🏆)

هذا المعالج هو **النموذج المعياري** الذي نريد تعميم أسلوبه على باقي التطبيق. التحسينات المطلوبة عليه لرفعه للمستوى «الخرافي»:

### الهوية البصرية
- ✅ **شاشة نجاح ذهبية بدل الخضراء** — رأس بتدرّج `gold-gradient`، علامة ✓ (SVG) داخل حلقة متوهّجة (`shadow-gold` + `ring`) مع أنيميشن `pop-in`. *(الكونفيتي مؤجّل — اختياري)*
- ✅ **كروت النوع (סוחר/קבלן) كـ Glass-cards** — خلفية زجاجية داكنة + حدّ ذهبي متوهّج عند الاختيار + أيقونة SVG مخصّصة (متجر/خوذة) بدل الإيموجي + علامة ✓ ذهبية بالزاوية + طفو عند hover.
- ✅ **Stepper متوهّج** — الخطوة الحالية بحلقة ذهبية نابضة (`animate-gold-pulse`)، والخط الواصل يتعبّأ بتدرّج ذهبي تدريجياً (animated fill) بدل القفزة اللونية.

### التجربة (UX)
- ✅ **ملخّص (Review) قبل الإنشاء** — بطاقة ذهبية بخطوة «תנאים» تعرض كل البيانات قبل الضغط على «יצירת חשבון».
- ✅ **مؤشر قوة كلمة المرور** (4 شرائح + تسمية)، وتحويل زر 🎲 لأيقونة refresh أنيقة (SVG).
- ✅ **معاينة حيّة لرسالة الواتساب** — فقاعة شات تعرض نص الرسالة قبل الإرسال بشاشة النجاح.
- ✅ **زر «نسخ الكل»** — «העתקת כל הפרטים» ينسخ الرابط+اليوزر+الباسوورد دفعة واحدة.
- ☐ **شريط تقدّم سفلي** «خطوة X من 4» بنسبة مئوية (للموبايل).

### لمسات «خرافية»
- ✅ انتقال fade ناعم بين الخطوات (`key={step}` + `animate-fade-up`).
- ✅ كروت تطفو قليلاً عند hover (`hover:-translate-y-0.5`).
- ✅ **QR code** لرابط الدخول بشاشة النجاح (مكتبة `qrcode`، يُولّد client-side).
- ✅ haptic/اهتزاز خفيف عند النجاح على الموبايل (`navigator.vibrate`).
- ✅ **كونفيتي ذهبي** لحظة الإنشاء (`<Confetti>` — CSS، بلا مكتبة).

## الجزء الثاني — تعميم أسلوب الـ Wizard على باقي التطبيق

فحص شامل لكل صفحات التطبيق (33 صفحة) لتحديد أين ينفع نطبّق نفس فكرة المعالج الموجّه المتكامل. مرتّبة حسب الأولوية والأثر.

### 🥇 مرشّحون قويون (تحويل لـ Wizard موصى به)

#### 1. محرّر المنتج — `/admin/products/edit` *(الأولوية القصوى)*
- **الحالة الحالية:** فورم واحد ضخم (~486 سطر، ~22 حقل عبر 4 أقسام قابلة للطي + ماسح باركود + صفوف مواصفات ديناميكية).
- **لماذا wizard:** الحقول تنقسم طبيعياً لمجموعات منطقية.
- **الخطوات المنفّذة:** (1) פרטים (اسم/slug/فئة/SKU/باركود/وصف) → (2) תמונה (صورة + datasheet) → (3) תמחור (تكلفة/أسعار/مخزون/علم) → (4) מפרט + سيخوم.
- ✅ **تم التحويل لمعالج** بنفس نمط `dealers/new` (stepper مشترك، انتقال ناعم، ملخّص، أزرار ذهبية). الـ stepper **قابل للنقر بوضع التعديل** (كل البيانات محمّلة).

#### 2. أوامر الشراء — `/admin/purchase-orders`
- **الحالة الحالية:** قائمة + فورم قابل للطي + لوحة «تجديد ذكي» (smart replenishment) — وهي عملياً شبه خطوة wizard جاهزة.
- **الخطوات المقترحة:** (1) اقتراحات التجديد الذكي (اختيار بالـ checkbox) → (2) المورّد + رقم الأمر → (3) مراجعة الأصناف → (4) إرسال.
- ☐ تحويل اللوحة الذكية لخطوة أولى رسمية في معالج.

#### 3. عروض الأسعار — `/admin/quotes`
- **الحالة الحالية:** قائمة + فورم داخل بطاقة قابلة للطي (اختيار عميل + أصناف ديناميكية + توصية سعر ذكية).
- **الخطوات المقترحة:** (1) اختيار العميل → (2) الشروط/التواريخ → (3) إضافة الأصناف (مع شريحة «💡 מומלץ») → (4) مراجعة وإرسال (+ زر واتساب لإرسال العرض).
- ☐ تحويل لمعالج موازٍ لأوامر الشراء.

### 🥈 مرشّح خفيف (اختياري — مكسب للموبايل)

#### 4. الدفع — `/checkout`
- **الحالة الحالية:** فورم بقسمين على صفحة واحدة (شحن + طريقة دفع) + ملخّص جانبي.
- **الخطوات المقترحة:** (1) تفاصيل الشحن → (2) طريقة الدفع → (3) مراجعة → تأكيد.
- ☐ تحسين تدريجي للموبايل (لكن لا يبطّئ مستخدم الديسكتوب الخبير — نقيّم لاحقاً).

## مكوّنات مشتركة مقترحة (لتوحيد الأسلوب)

لتجنّب تكرار الكود عند تعميم نمط الـ wizard، يُقترح استخراج مكوّنات قابلة لإعادة الاستخدام:

- ✅ `<WizardStepper>` — الـ stepper الذهبي المتوهّج المشترك + `<CheckIcon>` (يُستخدم في `dealers/new` و`products/edit`؛ يدعم النقر على الخطوات).
- ✅ `<Confetti>` — كونفيتي CSS بلا مكتبة (شاشات النجاح).
- ☐ `<GlassCard>` — بطاقة زجاجية داكنة بحدّ ذهبي (حالياً inline في كروت الاختيار — لم تُستخرج بعد).
- ☐ `<SuccessScreen>` — شاشة نجاح ذهبية موحّدة (حالياً inline في `dealers/new`).
- ☐ `<ReviewSummary>` — بطاقة الملخّص (حالياً `ReviewItem` مكرّر inline في الصفحتين — مرشّح للاستخراج).
- ☐ نقل أيقونات SVG المخصّصة لمجلد `components/icons`.
