مرجع كامل لأدوار المستخدمين في بوابة الطلبات B2B وما يقدر كل دور يعمله.

> المصدر الوحيد للدور: عمود `store.profiles.role` (`super_admin` / `admin` / `dealer`)

## طبقتا التحكّم

الصلاحيات مفروضة على مستويين — الواجهة للعرض، وقاعدة البيانات للأمان الحقيقي:

| الطبقة | أين | الوظيفة |
|--------|-----|---------|
| **الواجهة (UI)** | `useProfile()` ([`src/lib/auth.ts`](../src/lib/auth.ts)) · `AuthGuard` ([`src/components/AuthGuard.tsx`](../src/components/AuthGuard.tsx)) · `AdminLayout` ([`src/app/admin/layout.tsx`](../src/app/admin/layout.tsx)) | تقرّر ماذا يُعرض ومتى يُعاد التوجيه |
| **الأمان الحقيقي (DB)** | RLS على جداول `store.*` + دالة `store.is_admin()` + `public` views بـ `security_invoker = on` | ترفض أي وصول غير مصرّح حتى لو تلاعب أحدٌ بالواجهة |

دوال مساعدة أساسية في [`src/lib/auth.ts`](../src/lib/auth.ts):
- `isAdminRole(role)` → `true` لـ `admin` و `super_admin`.
- `canSeePrices(profile)` → `true` للموزّع `approved` أو أي أدمن.
- `useProfile()` يرجّع `{ profile, ready, showPrice, isSuperAdmin, … }`.

## هرم الأدوار

```
super_admin  ─►  كل شيء + إدارة المدراء والأدوار   (حصري: /admin/admins)
   admin     ─►  كل لوحة الإدارة                   (بدون إدارة أدوار)
 dealer ✅   ─►  كتالوج + أسعار + سلة + طلب + عروضه + بصمة   (بياناته فقط)
 dealer ⏳   ─►  مسجّل بانتظار الموافقة             (بدون أسعار/طلب)
  زائر       ─►  صفحة الدخول فقط
```

## 🔑 super_admin — المدير الرئيسي

يملك **كل قدرات الأدمن + ميزة حصرية:** إدارة المدراء والأدوار عبر `/admin/admins` ("🔑 מנהלים").

| القدرة | التفاصيل |
|--------|----------|
| إنشاء حساب مدير جديد | إيميل + باسوورد → Edge Function `admin-create-customer` ثم ترقية `role: admin` |
| ترقية موزّع → أدمن | "הענק הרשאות ניהול" |
| سحب صلاحيات أدمن → موزّع | "הסרת הרשאות" |
| عرض كل المستخدمين | قائمة كاملة بالأدوار والحالات |

- الرابط يظهر فقط لـ `isSuperAdmin` ([`admin/layout.tsx:43`](../src/app/admin/layout.tsx)).
- الصفحة ترفض غير السوبر أدمن ([`admins/page.tsx`](../src/app/admin/admins/page.tsx)).
- شارة **"מנהל ראשי"** ذهبية، ودوره **لا يُغيَّر** (ولا يغيّر دور نفسه).

> المدير العادي لا يقدر يغيّر الصلاحيات: *"מנהל יכול לגשת לכל ממשק הניהול, אך לא יכול לשנות הרשאות."*

## 🛡️ admin — المدير

يصل **كل لوحة الإدارة `/admin`** (الحماية: `AdminLayout` تتحقق عبر `is_admin()`). يرى الأسعار دائماً.

| القسم | المسار | الوظيفة |
|-------|--------|---------|
| סקירה | `/admin` | لوحة العمليات: KPIs، أرباح، COGS، مقارنة فترات، رسم بياني، توقّع نفاد مخزون، "נבץ היום" |
| הזמנות | `/admin/orders` | إدارة كل الطلبات |
| הצעות מחיר | `/admin/quotes` | عروض الأسعار + توصية سعر ذكية |
| לקוחות | `/admin/dealers` | الموزّعون + معالج إضافة تاجر/مقاول + إرسال الدخول واتساب + تيك الزبون 360° |
| מחירי לקוח | `/admin/customer-prices` | أسعار خاصة لكل زبون |
| מוצרים / קטגוריות | `/admin/products` · `/admin/categories` | الكتالوج، الأسعار، الباركود، التكلفة |
| מלאי | `/admin/inventory` | حركات المخزون + ماسح باركود |
| ספקים / הזמנות רכש | `/admin/suppliers` · `/admin/purchase-orders` | الموردون وأوامر الشراء |
| הגדרות | `/admin/settings` | إعدادات المتجر |

**الحد عن السوبر أدمن:** لا يخلق مدراء جدد ولا يغيّر أدوار المستخدمين.

## 👤 dealer — الموزّع / الزبون

التسجيل العام **مغلق** — الحساب يُنشأ حصراً من الأدمن (`/admin/dealers`). القدرات تعتمد على `status`:

### ⏳ dealer (pending) — بانتظار الموافقة
- يسجّل دخول، لكن **لا يرى الأسعار** ولا يقدر يطلب (`canSeePrices` → `false`). ينتظر موافقة الأدمن.

### ✅ dealer (approved) — مفعّل
| القدرة | التفاصيل |
|--------|----------|
| تصفّح الكتالوج | `/products` + رؤية أسعار الجملة (وأسعاره الخاصة إن وُجدت عبر `applyEffectivePrices`) |
| ماسح باركود/QR | وصول سريع للمنتج |
| السلة والطلب | `/cart` → `/checkout`. ⚠️ السعر يُحسب على السيرفر عبر `store.place_order` (SECURITY DEFINER) — **العميل لا يحدّد السعر أبداً** |
| الحساب الشخصي `/account` | هَب "האזור האישי": الطلبات + عروض الأسعار + الأمان/البصمة + خروج |
| طلباتي | `/account/orders` — السجل + إعادة طلب (`resolveReorder`) |
| عروض الأسعار | `/account/quotes` — وقبول عرضه وتحويله لطلب (`accept_my_quote`) |
| الأمان/البصمة | `/account/security` — passkeys متعددة الأجهزة + إشعارات Push |

**الحدود:** أي وصول لـ `/admin` → شاشة **"אין הרשאה"**. وRLS تمنعه من رؤية بيانات أي زبون آخر — كلٌّ يرى **بياناته فقط**.

## 🚪 الزائر (غير مسجّل)

- **صفحة الدخول `/` فقط.** أي مسار آخر → `AuthGuard` يعيد التوجيه فوراً لـ `/?redirect=<path>` ([`AuthGuard.tsx:27`](../src/components/AuthGuard.tsx)).
- الدخول بالإيميل أو الـ username (عبر `get_email_by_username`) أو البصمة.
